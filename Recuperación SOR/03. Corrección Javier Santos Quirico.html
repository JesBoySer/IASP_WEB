1.2 Esquema de Particionado (Servidor con 4GB RAM y 40GB Disco)

Respuesta del Alumno Sin respuesta.
Solucionario Oficial
  • Partición 1 (SWAP): Tamaño: 8 GB (RAM x 2). FS: swap. Montaje: [swap]. Actúa como memoria virtual.
  • Partición 2 (Raíz /): Tamaño: 9,6 GB (30% del resto). FS: ext4. Montaje: /. Aloja el sistema operativo.
  • Partición 3 (Datos /home): Tamaño: 22,4 GB (70% del resto). FS: ext4. Montaje: /home. Protege los datos de usuarios ante llenados de disco.
Riesgo de usar /dev/sdb1 frente a UUID: Los nombres de dispositivo se asignan dinámicamente por el kernel. Al añadir hardware nuevo, el orden puede cambiar y el sistema fallaría al arrancar o corrompería datos. El UUID es inmutable.
Línea fstab corregida: 
UUID=a1b2c3d4-1234-5678-abcd-ef0123456789 /datos ext4 defaults 0 2

2.1 Conceptos Fundamentales de Identidades

Respuesta del Alumno
  • Usuario: Persona con una cuenta identificada por su nombre que realiza operaciones según permisos asignados.
  • Grupo: Donde se agrupan los usuarios, organizados en su carpeta y con sus permisos.
  • UID / GID: Número utilizado para identificar a los usuarios (en ambos casos).
  • Ventajas: Permite dar permisos de manera rápida y automatizada en bloque, y facilita la búsqueda de usuarios por organización de carpetas.
Solucionario Oficial
  • Usuario: Entidad que representa a una persona o proceso. Posee un login único y UID.
  • Grupo: Conjunto lógico de usuarios para asignación colectiva de accesos. Posee un GID único.
  • UID: User Identifier numérico utilizado internamente por el kernel de Linux. (Root = 0).
  • GID: Group Identifier numérico que determina los accesos grupales del kernel.
  • Ventajas: Centralización, mantenimiento reducido al mover usuarios de grupo sin tocar recursos, consistencia y menor probabilidad de errores.

2.3 Estrategia de Anidamiento de Grupos (AGDLP)

Respuesta del Alumno "El grupo global guarda los usuarios y el local de dominio guarda los grupos globales... Lo hicimos en clase metiendo dentro de finanzas_rwx al grupo gerente, de manera que al aplicar setfacl le das permisos a todos los usuarios a la vez."
Solucionario Oficial Estructura: Accounts → Global Groups (roles de función) → Domain Local Groups (permisos sobre el recurso) → Permissions. Separar identidades de accesos dota al sistema de escalabilidad ilimitada, facilitando auditorías y minimizando errores catastróficos de privilegios.

RA 4: Gestión de recursos compartidos en red y seguridad perimetral local (Nota: 6.5 / 10)

4.1 Permisos, Derechos y Caso Práctico Cruzado

Respuesta del Alumno
  • Permiso: Acciones que un usuario puede hacer sobre un archivo/carpeta.
  • Derecho: Partes del sistema a las que puede acceder el usuario.
  • Caso práctico: El usuario NO podrá entrar. Aunque tenga control total sobre los archivos, se bloquea antes en la primera puerta al no tener derecho de acceso desde la red.
Solucionario Oficial
  • Permiso vs Derecho: El permiso aplica al objeto (ACL). El derecho aplica a la directiva global del sistema.
  • Resolución: Correcto. El derecho "Acceder a este equipo desde la red" se evalúa en el proceso de autenticación previo. Si falta, la conexión se rechaza inmediatamente sin llegar a leer los permisos NTFS.

4.2 Permisos de Servidor de Archivos (ACLs y smb.conf)

Respuesta del Alumno El alumno definió rutas relativas erróneas en sus comandos e intentó asignar permisos a grupos locales simples de Linux en lugar de las identidades del dominio Active Directory. 
sudo setfacl -R -m g:DL_Direccion_rwx:rwx /srv/samba/proyectos
En smb.conf mapeó los recursos de la siguiente forma: 
[proyectos]
path = /srv/samba/proyectos
read only = no
valid users = @DL_Proyectos_rwx, @DL_Proyectos_rx
Solucionario Oficial Se deben aplicar ACLs extendidas relativas al dominio y configurar las herencias por defecto (-d): 
setfacl -m g:"RECUPERACION\DL_Proyectos_RW":rwx /srv/samba/proyectos
setfacl -d -m g:"RECUPERACION\DL_Proyectos_RW":rwx /srv/samba/proyectos
Configuración en smb.conf: 
[proyectos]
path = /srv/samba/proyectos
read only = no
browseable = yes
valid users = @"RECUPERACION\Domain Users"
create mask = 0660
directory mask = 0770
