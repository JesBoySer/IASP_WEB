Solución de Referencia

Fallo grave de estructura YAML. Se requería direccionamiento estático (no DHCP) y la estructura anidada de DNS es nameservers: addresses:. Añadir máscaras de red (/24) a las IPs de DNS invalida el fichero.

  renderer: networkd
  ethernets:
    enp0s3:
      dhcp4: no
      addresses:
        - 192.168.100.10/24
      routes:
        - to: default
          via: 192.168.100.1
      nameservers:
        addresses:
          - 192.168.100.10
          - 8.8.8.8

RA 2: Configuración y Gestión de Servicios de Directorio LDAP (Nota: 6.5/10)
2.1 Fundamentos y Conceptos de Identidades de Red

Respuesta del Alumno

Usuario/Grupo: Entidades con UID y conjuntos de usuarios con un GID primario. Permite centralizar la gestión de los privilegios.

UID/GID: Indica que son "la información que hay de un usuario" y "el conjunto de permisos que tiene el grupo".

Solución de Referencia

Las definiciones de UID y GID son coloquiales e inexactas. Conceptualmente son los identificadores numéricos unívocos utilizados por el kernel de Linux para validar los privilegios sobre ficheros basándose en la identidad, no el almacén de texto de la información del usuario.

RA 3: Integración de Sistemas Heterogéneos - Samba AD DC (Nota: 2.5/10)
3.1 Comprensión de la Nomenclatura del Árbol de Directorio

Respuesta del Alumno

CN: Nombre de usuario. | OU: Grupo al que pertenece. | DC: "Domain Control" (Controlador de dominio).

Dominio detectado: salapet

Solución de Referencia

Errores conceptuales graves: OU significa Unidad Organizativa (contenedor lógico, no grupo). DC significa Domain Component (Componente de Dominio), no Domain Control. El dominio FQDN real de los metadatos es salapet.local (o salapet.es según el bloque anterior), nunca la etiqueta única.

3.2 Script de Automatización de Infraestructura Active Directory (samba-tool)

Respuesta del Alumno

sudo samba-tool ou add "OU=Comun,DC=samba,DC=srv"
...
sudo samba-tool ou add "OU=Comun,OU=Grupos,DC=samba,DC=srv"
...
sudo samba-tool group add G_Dirección "OU=Comun,OU=Grupos,DC=samba,DC=srv"
sudo samba-tool group add G_Dirección "OU=Proyecto,OU=Grupos,DC=samba,DC=srv"

Solución de Referencia

El script presenta fallos insalvables de ejecución:

  1. El subcomando de creación en Samba es samba-tool ou create, no add.
  2. Rompe la precedencia lógica: intenta crear OUs hijas dentro de "OU=Grupos" o "OU=Usuario" antes de haber creado dichos contenedores padre.
  3. Duplica los comandos de creación del mismo nombre de grupo en diferentes OUs ficticias (OU=Proyecto en singular en lugar de OU=Proyectos), lo cual causaría colisión de SID/SAM y fallos de sintaxis críticos.
RA 4: Compartición de Recursos y Políticas de Seguridad (Nota: 0.5/10)
4.1 Teoría Avanzada de Seguridad: Permiso vs Derecho

Respuesta del Alumno

Permiso: Es cuando alguien necesita hacer algo para un momento (Ej. Control total temporal a un informático).

Derecho: Es darle a alguien el poder de hacer algo cuando quiera (Ej. Permiso de modificar fijo).

Caso Práctico (Control Total sin Derecho de Red): [En blanco]

Solución de Referencia

Explicación errónea. Un permiso se aplica a un recurso físico u objeto del sistema de archivos (DACL/ACL). Un derecho o privilegio es una directiva asignada a una cuenta para realizar acciones en el ámbito total del sistema operativo (ej. Iniciar sesión local, apagar el equipo).

Resolución del Caso: El usuario NO puede conectarse a la red. El derecho "Acceder a este equipo desde la red" se evalúa en la capa previa de autenticación del sistema operativo. Si falta dicho derecho, la conexión SMB se rechaza inmediatamente y los permisos NTFS/ACL nunca llegan a evaluarse.

5.1 Comando de Monitorización y Diagnóstico de Rendimiento

Respuesta del Alumno

Tráfico de Red: [En blanco]

Actividad de Disco: top (Justificación: "Se puede ver").

Logs Críticos: journalctl -p crit -b (Correcto).

Solución de Referencia

El uso de top para evaluar cuellos de botella en operaciones de entrada/salida (I/O) de disco es incorrecto. La herramienta idónea y requerida para medir transferencias en tiempo real de disco por proceso es iotop, y para red es nethogs o iftop.

5.2 Análisis Técnico de Métricas de Recursos (Muestra free -m)

Respuesta del Alumno

RAM/SWAP: Indica que consume todo porque tiene un virus minando criptomonedas o muy poca capacidad.

Load Average (En máquina con 2 núcleos): "La media de tiempo de uso de cada núcleo desde que se encendió la máquina por primera vez".

Solución de Referencia

Fallo conceptual severo. El Load Average no calcula medias de tiempo históricos desde el arranque; es un indicador dinámico que mide la cantidad de procesos en cola de ejecución o bloqueados en I/O en intervalos de 1, 5 y 15 minutos. Un valor de 5.81 en una máquina de 2 núcleos indica una saturación extrema inmediata del sistema (~2.9 veces por encima de la capacidad nominal de procesamiento).

5.3 Gestión de Repositorios, Índices y Paquetes (APT)

Respuesta del Alumno

Fallo crítico por inversión absoluta de los conceptos:

Comando para actualizar índices: sudo apt upgrade

Comando para instalar actualizaciones: sudo apt update

Solución de Referencia

La inversión total de los comandos haría imposible la administración del servidor de producción. update descarga metadatos/índices desde los repositorios y upgrade aplica las descargas e instalaciones de las versiones superiores del software.

5.4 Filtrado de Bitácoras Avanzado (Journalctl Pipelines)

Respuesta del Alumno

Resolución aceptable mediante entubado genérico de texto:

  • A) journalctl -b | grep "kernel"
  • B) journalctl -b | grep "ssh"
  • C) journalctl --since=yesterday --until=today | grep "cron"

Solución de Referencia

Las respuestas del alumno funcionan pero consumen muchos recursos de CPU. Se debían priorizar los flags nativos del sistema de journald, reduciendo el parseo de cadenas:

  • A) journalctl -b -k -p err (Nativo para kernel)
  • B) journalctl -u ssh -f (Seguimiento en vivo por Unit)
  • C) journalctl -u cron --since yesterday --until today
